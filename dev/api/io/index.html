SLAM.Visualizer — Type

Visualizer for the SLAM.

Can be used to play in "live" mode as the SLAM is working or to replay from ReplaySaver.

Usage:

To use it in "live" preview mode, pass it as a keyword argument, when creating SlamManager:

visualizer = Visualizer(;resolution=(900, 600), image_resolution=(370, 1226))
SlamManager(params, camera; visualizer)

set_frame_wc!(v::Visualizer, frame_id, wc)

Add new frame wc to the visualizer queue. This is used when other threads are updating the visualizer.

The queue is processed in process_frame_wc! method.

SLAM.ReplaySaver — Type
ReplaySaver()

ReplaySaver is used to save/load SLAM result for the visualizer. If added to the SlamManager instead of the visualizer, it accumulates poses needed for the visualization/replay and can be later used to replay the results.

Usage:

To use it, pass it to visualizer keyword argument, when creating SlamManager. Do some work, and in the end, save it to the directory.

saver = ReplaySaver()
SlamManager(params, camera; visualizer=saver)
# ...run slam...
SLAM.save(saver, "save_dir")

Then you can load it from the save directory and use it to replay results.

SLAM.load!(saver, "save_dir")
